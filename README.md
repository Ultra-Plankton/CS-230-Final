# CS-230-Final
Final Assignment for CS-230


1. Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?

The client was The Gaming Room, the company behind the game Draw It or Lose It. They wanted a web-based version of their existing Android game that could run across multiple operating systems and devices. The software needed to support multiple games, teams, and players while ensuring unique names and maintaining a single GameService instance using the Singleton design pattern. The application also needed to be scalable, secure, and easy to maintain.

2. What did you do particularly well in developing this documentation?

I believe I did a good job explaining the software requirements and recommending technologies that met the client's needs. I clearly justified why Linux, Java, and a client-server architecture were appropriate choices and incorporated object-oriented design principles such as inheritance, encapsulation, and design patterns into the solution.

3. What about the process of working through a design document did you find helpful when developing the code?

Working through the design document helped me organize the project before writing any code. It allowed me to think through the system architecture, class relationships, and design patterns in advance, making the implementation process more structured and reducing the likelihood of major design changes later.

4. If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

If I could revise one section, I would expand the system architecture portion. Although it was not required for the project, I would include diagrams showing how the client, web server, application server, and database communicate. This would provide a clearer understanding of the overall system design.

5. How did you interpret the user's needs and implement them into your software design? Why is it so important to consider the user's needs when designing?

I interpreted the client's requirements by identifying the key features they needed, including cross-platform compatibility, support for multiple users, unique game and team names, and reliable performance. I implemented these needs through object-oriented design, the Singleton and Iterator patterns, and a scalable client-server architecture. Considering the user's needs is important because the software must solve the client's problems while remaining easy to use, reliable, and capable of future growth.

6. How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

I approached the design by first analyzing the client's requirements and then selecting technologies and design patterns that best addressed those needs. I used object-oriented principles, UML modeling, and evaluations of different operating systems to guide my decisions. In the future, I would continue using requirement analysis and UML diagrams while also gathering additional user feedback early in the design process and creating prototypes to validate design decisions before implementation.
